⚠️ 反射型 XSS 漏洞演示

发生了什么？

你刚才在 URL 中注入了 JavaScript 代码，页面未经转义直接将其渲染到 DOM 中，导致恶意脚本执行。

// 漏洞代码（不安全） document.getElementById('searchKeyword').innerHTML = userInput; // 修复方案（安全） document.getElementById('searchKeyword').textContent = userInput;

攻击原理：攻击者构造含恶意脚本的 URL，诱导受害者点击。服务器将用户输入"反射"回页面，浏览器执行注入的脚本。
